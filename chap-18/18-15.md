# 18-15

O protocolode isolamento de *snapshot* utiliza uma etapa de validação que, antes de realizar uma gravação de um item de dados pela transação $T$, verifica se uma operação concorrente com $T$ já escreveu o item de dados.

**a)** Uma implementação simples usa um *timestamp* inicial e um *timestamp* de  confirmação para cada transação, além de um *conjunto de atualização*, que é o conjunto de itens de dados atualizados pela transação. Explique como executar a avalidação no esquema "primeiro confirmador vence" usando os *timestamps* de transação juntamente com os conjuntos de atualização. Você pode assumir que a etapa de validação e outras etapas de processamento de confirmação são executadas em série, isto é, para uma transiação de cada vez.

**b)** Explique como a etapa de validação pode ser implementada como parte do processamento de confirmação no esquema "primeiro confirmador vence", usando uma modificação no esquema anterior, no qual, em vez de usar conjuntos de atualização, cada item de dados tem de escrever um *timestamp* associado a ele. Novamente aqui você pode assumir que a etapa de validação e outras etapas de processamento de confirmação são executadas em série.

**c)** O esquema do "primeiro atualizador vence" pode ser implementado usando *timestamps* como descrito antes, exceto que a validação é feita imediatamente após a aquisição de um bloqueio exclusivo, em vez de ser feito no momento da confirmação.

&emsp;**i.** Explique como atribuir *timestamps* de escrita para itens de dados para implementar o esquema do primeiro atualizador vence.

&emsp;**ii.** Mostre que, como resultado do bloqueio, se a validação é repetida no momento da confirmação, o resultado não mudará.

&emsp;**iii.** Explique por que não há necessidade de realizar a validação e outras etapas de processamento de confirmação em série nesse caso.
