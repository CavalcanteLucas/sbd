# 16-01

Faça o *download* do esquema de banco de dados e do grande *dataset* de universidade em `dbbook.com`. Crie o esquema da universidade no seu banco de dados favorito e carregue o *dataset*. Use o recurso **explain** descrito na Nota 16.1 para visualizar o plano escolhido pelo banco de dados em diferentes casos, conforme detalhado a seguir.

**a)** Escreva uma consulta com uma condição de igualdade sobre $student.name$ (que não tem índice) e veja o plano escolhido.

**b)** Crie um índice sobre o atributo $student.name$ e veja o plano escolhido para a consulta do item (a).

**c)** Crie consultas simples juntando duas ou três relações e veja os planos escolhidos.

**d)** Crie uma consulta que calcule uma agregação com agrupamento e veja o plano escolhido.

**e)** Crie uma consulta SQL cujo plano escolhido use uma operação de semijunção.

**f)** Crie uma consulta SQL que use uma cláusula **not in**, com uma subconsulta usando agregação.

**g)** Crie uma consulta para a qual o plano escolhido use avaliação correlacionada (o modo como a avaliação correlacionada é representada depende do banco de dados, mas a maioria deles mostraria um filtro ou um operador de proejeção com um subplano ou uma consulta).

**h)** Crie uma consulta de atualização SQL que atualize uma única linha em uma relação. Veja o plano escolhido para essa consulta de atualização.

**i)** Crie uma consulta de atualização SQL que atualize grande número de linhas em uma relação, usando uma subconsulta para calcular o novo valor. Veja o plano escolhido para essa consulta de atualização.
